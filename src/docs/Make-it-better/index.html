Make your H5BP-project better

Here are some useful advices how you can make your project with HTML5 Boilerplate even better. We don't want to include them as they do not fit for everyone and are more like an addition.

JS

  • Use a script loader like yepnope.js in combo with your HTML5 polyfills to bring the future to all browsers.
    • yepnope.js is already part of Modernizr. So you can use it right away.
  • Tell your users to Enable JavaScript (maybe with a link to enable-javascript.com)
  • Simulate link printing in IE with JS
  • Kick off your page-specific javascript based on your markup.

CSS

  • Consider CSS3please.com to assist in writing CSS3
  • Use css3pie.com as a CSS3 polyfill
  • Read some CSS Spriting tips or use Lemonade if you are using Sass
  • Color considerations for printing
  • Using border-radius with a border in Webkit? Try -webkit-background-clip to fix the bleed.
  • Write efficient CSS for optimum browser performance.

  • Know your available browser CSS hacks.

  • Gecko-based browsers (Firefox, Flock) have a outline bug on buttons(demo). The solution is to add:
    button:focus { outline-width:1px; outline-style:dotted; } button::-moz-focus-inner { padding:0; border:0; }
    (Thanks Ivannikolic)

  • Fieldset nested in a list item bug in Firefox - description, test case and fix.
  • Style input placeholders with vendor-prefixed extensions
  • If hr has a width, make sure you have text-align: left also set for consistent rendering across all browsers. IE8 renders the hr in the center while others align it to the left. (see #297)
  • Consider less visually intese colors for text selection when the window is inactive.
  • If you're building a web app you may want native style momentum scrolling in iOS5 using -webkit-overflow-scrolling: touch.
  • Using text-decoration: underline; for anchors cause some issues specially for languages that have dots under some letters. Use border-bottom: 1px solid currentcolor; in such cases. Note: consider that text-shadow does not effect borders.
  • You can add hyphen to hyphenate your text, read more here
  • Consider not setting :hover states for touch devices

HTML

  • Use HTML5 polyfills
  • Learn HTML5 elements better: HTML5 Doctor Glossary and Dive Into HTML5 : semantic elements

  • You can use the body element as a wrapper, so you need not use a div#container.

  • Use rel="prefetch" for documents to pre-fetch.

  • Use Microformats (via microdata) for optimum search results visibility

Head

  • More meta tag tracks for your <head>

  • if you're a photo gallery site you might want this for ie6 to not show that stupid thing on <img> tags:

    <!--[if IE]><meta http-equiv="imagetoolbar" content="no" /><![endif]-->

  • To remove gloss from iPhone home screen rel value should be "apple-touch-icon-precomposed"

  • How to make Android recognize Apple Touch Icons
  • Use the favicon PSD template to generate all of the touch icons
  • Read more on DNS-Prefetching
  • Use table-layout: fixed to make large tables render faster

.htaccess

  • If you can, implement raw DEFLATE compression instead of GZIP; DEFLATE is faster and always results in a smaller output. Deflate compression browser compatibility and advantages over GZIP. (Note: Apache's mod_deflate outputs GZIP)
  • Using trailing-slash-redirects Proper usage of trailing slash redirects

Misc

  • Use Google's recommendations for SEO: the Google SEO Starter Guide

  • Dig into the new link relations

  • Consider some opinionated web do's and web dont's

  • Get your team hooked using the same front-end coding standards and best practices.
  • What should a developer know before building a public web site? Stackoverflow great summaries.
  • Read more on Common-Analytics-Augments
  • Screen readers currently have less-than-stellar support for HTML5 but the JS script accessifyhtml5.js can help increase accessibility by adding ARIA roles to HTML5 elements

Using a mobile-first approach

Consider setting your base styles and mobile-friendly less-image-heavy styles first, and use media queries to set the more intensive desktop friendly styles. Our 2.0 release includes respond.js within the modernizr.js and enables all media-queries to work in IE6, 7 and 8.
